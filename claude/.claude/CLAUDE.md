## Overall
Use red/green TDD

When code needs explanation, prefer brief inline comments in the source file over expanding README/docs.

## Commit Message Standards
When writing commit messages:
- Use imperative mood (e.g., "Add", "Fix", "Update", "Refactor") 
- Be objective, accurate, and concise
- NEVER include self-references or attribute code to yourself: "I", "me", "Claude", "by Claude"
